# Календарь вкладов

Одностраничное приложение для учёта вкладов и контроля сроков закрытия. Приложение работает без сервера: каждый пользователь хранит свои данные только в `localStorage` своего браузера.

## Приватность данных

- Вклады не записываются в файлы проекта, GitHub, `dist` или на сервер.
- У каждого человека данные отдельные, потому что `localStorage` изолирован браузером, профилем пользователя и доменом сайта.
- Если другой человек откроет опубликованную ссылку, он увидит собственное локальное хранилище, а не ваши вклады.
- При первом запуске приложение загружает тестовые демо-вклады. Это примеры из кода, не реальные пользовательские данные.
- Очистка данных сайта в браузере удалит локально сохранённые вклады для этого браузера.

## Локальная разработка

```bash
npm install
npm run dev
```

## Проверка

```bash
npm test
npm run build
```

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Загрузите в него исходный код проекта.
3. В настройках репозитория откройте `Settings -> Pages`.
4. В `Build and deployment` выберите `GitHub Actions`.
5. Запушьте изменения в ветку `main`.
6. Workflow `Deploy to GitHub Pages` соберёт приложение и опубликует содержимое `dist`.

После публикации приложение будет доступно по адресу вида:

```text
https://<username>.github.io/<repo>/
```

Эту ссылку можно передавать другим людям. Их вклады будут сохраняться только у них в браузере.

## Что не нужно публиковать вручную

- `node_modules/`
- `dist/`
- `*.tsbuildinfo`
- сгенерированные файлы `vite.config.js` и `vite.config.d.ts`

Эти файлы исключены через `.gitignore`; GitHub Actions создаст всё необходимое при деплое.
