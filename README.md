# Календарь вкладов

Одностраничное приложение для учёта вкладов и контроля сроков закрытия. Приложение работает без сервера: в браузере данные хранятся в `localStorage`, а в desktop-версии — в локальном JSON-файле на компьютере пользователя.

## Приватность данных

- Вклады не записываются в файлы проекта, GitHub, `dist` или на сервер.
- У каждого человека данные отдельные, потому что `localStorage` изолирован браузером, профилем пользователя и доменом сайта.
- Если другой человек откроет опубликованную ссылку, он увидит собственное локальное хранилище, а не ваши вклады.
- При первом запуске приложение загружает тестовые демо-вклады. Это примеры из кода, не реальные пользовательские данные.
- Очистка данных сайта в браузере удалит локально сохранённые вклады для этого браузера.

```bash
npm install
npm run dev
```

```bash
npm test

```

## Desktop-версия для Windows и macOS

Desktop-версия собирается через Electron и использует тот же React-интерфейс. В desktop-режиме вклады хранятся не в `localStorage`, а в JSON-файле в системной папке приложения.

Примерные пути хранения:

```text
Windows: C:\Users\<user>\AppData\Roaming\Vklad Tracker\deposits.json
macOS: ~/Library/Application Support/Vklad Tracker/deposits.json
```

Это значит, что переустановка или обновление приложения не удаляет вклады пользователя.

Локально открыть desktop-версию:

```bash
npm run desktop:dev
```

Собрать папку приложения без установщика:

```bash
npm run desktop:pack
```

Собрать установщики:

```bash
npm run desktop:build
```

Результат сборки появится в папке `release/`. Для macOS собирается `.dmg`, для Windows настроен NSIS-установщик. Обычно macOS-установщик нужно собирать на Mac, а Windows-установщик удобнее собирать на Windows или в CI.

Обновление приложения можно делать вручную: пользователь скачивает новую версию установщика и устанавливает поверх старой. Его файл `deposits.json` остаётся в системной папке приложения.

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Загрузите в него исходный код проекта.
3. В настройках репозитория откройте `Settings -> Pages`.
4. В `Build and deployment` выберите `GitHub Actions`.
5. Запушьте изменения в ветку `main`.
6. Workflow `Deploy to GitHub Pages` соберёт приложение и опубликует содержимое `dist`.

После публикации приложение будет доступно по адресу вида:

```text
https://<username>.github.io/<repo>/
```

Эту ссылку можно передавать другим людям. Их вклады будут сохраняться только у них в браузере.

## Что не нужно публиковать вручную

- `node_modules/`
- `dist/`
- `*.tsbuildinfo`
- сгенерированные файлы `vite.config.js` и `vite.config.d.ts`

Эти файлы исключены через `.gitignore`; GitHub Actions создаст всё необходимое при деплое.
